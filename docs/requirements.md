# PrimiHub 需求规格说明书

## 1. 项目背景

### 1.1 行业背景

随着数字经济的持续高速增长，数据的互联互通需求越来越旺盛。从政府机关的机密数据、公司核心商业数据到个人信息，数据的流通需求无处不在。然而，近年来我国相继出台了《数据安全法》和《个人信息保护法》，对数据的合规使用提出了严格要求。如何在保护数据隐私的前提下实现数据的价值流通，成为亟需解决的关键问题。

隐私计算技术作为连接数据流通和隐私保护法规的纽带，实现了"数据可用不可见"——即在保护数据本身不对外泄露的前提下实现数据分析计算的技术集合。隐私计算已广泛应用于金融、医疗、通信、政务等多个行业。

### 1.2 项目定位

PrimiHub 是一个由密码学专家团队打造的开源隐私计算平台，旨在提供安全可靠、开箱即用、自主研发、功能丰富的隐私计算解决方案。平台覆盖安全多方计算（MPC）、联邦学习（FL）、隐私集合求交（PSI）、隐匿查询（PIR）、同态加密（HE）、可信执行环境（TEE）等主流隐私计算技术。

## 3. 功能需求

### 3.1 隐私集合求交 (PSI)

| 需求 ID | 功能描述 | 优先级 |
|---------|---------|--------|
| PSI-01 | 支持 ECDH 协议的隐私集合求交 | P0 |
| PSI-02 | 支持 KKRT 协议的高效隐私集合求交 | P0 |
| PSI-03 | 支持基于 TEE 的隐私集合求交 | P1 |
| PSI-04 | 支持交集（INTERSECTION）和差集（DIFFERENCE）两种模式 | P0 |
| PSI-05 | 支持指定参与方的数据列进行求交 | P0 |
| PSI-06 | 支持结果同步到服务端 | P1 |
| PSI-07 | 支持去重选项 | P1 |

**用例：多机构黑名单联合查询**

```
参与者：银行A（Client）、银行B（Server）
前置条件：双方已部署 PrimiHub 节点
基本流程：
1. 银行A 准备客户黑名单数据 CSV 文件
2. 银行B 准备客户黑名单数据 CSV 文件
3. 发起方通过 CLI 提交 PSI 任务
4. 双方节点执行 ECDH 隐私求交协议
5. 返回交集结果到发起方
后置条件：双方获得共同的黑名单客户列表，但不知道对方其他客户信息
```

**用例：跨医院联合疾病预测**

```
参与者：医院A（Host）、医院B（Guest）
前置条件：双方有重叠的患者样本，持有不同的特征

1. 双方注册各自的数据集到元数据服务
2. 发起方提交纵向 XGBoost 训练任务
3. 双方通过 PSI 对齐样本
4. 执行纵向联邦学习训练
5. 保存模型到指定路径
```

## 6. 术语表

| 术语 | 英文 | 定义 |
|------|------|------|
| 隐私求交 | PSI (Private Set Intersection) | 多方在不泄露各自数据的前提下计算数据交集 |
| 隐匿查询 | PIR (Private Information Retrieval) | 用户在不暴露查询内容的前提下检索数据库 |
| 联邦学习 | FL (Federated Learning) | 多方在不共享原始数据的前提下联合训练模型 |
| 安全多方计算 | MPC (Secure Multi-Party Computation) | 多方在不泄露各自输入的前提下共同计算函数 |
| 同态加密 | HE (Homomorphic Encryption) | 直接对加密数据进行计算，无需解密 |
| 可信执行环境 | TEE (Trusted Execution Environment) | 基于硬件的安全隔离环境 |
| 差分隐私 | DP (Differential Privacy) | 在查询结果中添加噪声以保护个体隐私 |
| 横向联邦 | HFL (Horizontal Federated Learning) | 特征相同、样本不同的联邦学习模式 |
| 纵向联邦 | VFL (Vertical Federated Learning) | 样本相同、特征不同的联邦学习模式 |
| 元数据服务 | Meta Service | 管理数据集注册、发现和协作的服务 |
